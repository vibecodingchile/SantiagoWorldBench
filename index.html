Abstract

TL;DR

We present Santiago World Model (SWM-SCL), a video world model that generates photorealistic, navigable footage of Santiago de Chile anchored to real street geometry. Unlike generative world models that hallucinate environments from scratch, SWM-SCL retrieves Mapillary street-view imagery near the target GPS coordinates and uses it as geometric and semantic conditioning for a Cosmos-Predict2.5-2B DiT backbone fine-tuned with LoRA.


Our method adapts four key techniques from the Seoul World Model (Seo et al., 2025): Cross-Temporal Pairing to prevent copying of dynamic objects, a Virtual Lookahead Sink to stabilize autoregressive drift, Intermittent Freeze-Frame for smooth view interpolation across sparse keyframes, and joint Geometric + Semantic Referencing via depth splatting and token injection. Critically, we replace NAVER's proprietary map data with Mapillary (open API), making the full pipeline reproducible by any researcher with a free token.


We introduce SantiagoWorldBench, a benchmark of 150 curated test trajectories across 5 urban zones, evaluating video quality, camera trajectory fidelity, and street-geometry alignment against GPS ground truth.

Method

SWM-SCL extends the Seoul World Model to Santiago using open Mapillary data. The pipeline takes GPS coordinates, camera trajectory, and a text prompt — retrieves nearby street-view references — and generates temporally consistent video.

TECHNIQUE 01

Cross-Temporal Pairing

≥30 days gap

References retrieved from timestamps ≥30 days apart from the target sequence. Forces the model to learn persistent urban geometry (buildings, sidewalks) while ignoring transient content (vehicles, pedestrians).

TECHNIQUE 02

Virtual Lookahead Sink

Δ_VL = 5 frames

Dynamically retrieves the street-view image nearest to the endpoint of each chunk and injects it as a "virtual future frame", re-anchoring generation to real geometry at each autoregressive step.

TECHNIQUE 03

Intermittent Freeze-Frame

VAE stride = 4

Mapillary keyframes captured every 5–20m. The 3D VAE compresses 4 frames into 1 latent. We repeat each keyframe ×4 before encoding and discard repetitions on decode — no architecture changes required.

TECHNIQUE 04

Geometric + Semantic Ref

Dual conditioning

Geometric: depth-based forward splatting of reference to target viewpoint. Semantic: CLIP token injection into cross-attention provides appearance and texture priors from the street-view reference.

Left: Cross-Temporal Pairing prevents copying of dynamic objects. Right: Virtual Lookahead Sink sequence — Z_seq = [Z_hist ; Z_target ; z_VL].
